RS触发器

我们之前看到的东西就是RS触发器 (Reset-Set Trigger)，它貌似是触发器中最基础的一个。

如图，它由两个与非门组成（貌似可以用或非门实现但是这里不讨论）。

输入端有置0端(R)和置1端(S)。

输出端有Q和Q’（Q’在本文外一般写作 $ \bar Q $ 只是打上划线不方便所以懒得打233）。

来分析一下（我们再偷个懒把 $\bar { Q }$ 打成 $ Q’ $ (不然会和非运算产生歧义)）：

Q在 $ n + 1 $ 时刻的状态 $ Q{n + 1} = \overline{Q{n}’ \cdot S_{n}}$

而 $ Q{n}’ = \overline{Q{n} \cdot R_{n}} $

就得到递推关系： $ Q{n + 1} = Q{n} \cdot R_{n} + \bar{S_n} $

写个状态表：

Input Output
 Function
R S Qn Qn+1
0 0 0 x Undefined
1 x
0 1 0 0 Set status 0
1 0
1 0 0 1 Set status 1
1 1
1 1 0 0 Keep status
1
 1

发现其中R=S=0时输出未定义，没有在网上找到比较好的解释。

有两种说法：操作同时将Q置为0和1；操作使Q和Q’不互补。
